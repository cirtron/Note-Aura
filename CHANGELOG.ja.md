# 変更履歴

Note-Aura の重要な変更はすべてここに記録されています（最新順）。

## 2026-06-27

### 追加
- **管理者：IP アドレスブロック。** 管理者は新しい **管理 → ブロック済み IP** ページから IP アドレスをブロックできるようになりました。ブロックされた IP はセッション解決前のミドルウェアレベルで HTTP 403 で拒否されます。ブロックリストにはオプションの理由メモを追加でき、各 IP がブロックされた日時も表示されます。ブロック解除は即時反映されます。
- **管理者：ログインロックアウト設定。** 設定した回数の連続ログイン失敗後、アカウントが設定した分数ロックされます。両方の閾値は **管理 → ログインロックアウト** で設定します（`login_max_attempts`、デフォルト 5；`login_lockout_minutes`、デフォルト 15）。ログイン成功時にカウンターはリセットされます。管理者はユーザーページから任意のユーザーのロックアウトを解除できます。
- **管理者：強制ログアウトと時限停止。** ユーザーページに **強制ログアウト** ボタンが追加され、ユーザーのすべてのアクティブなセッションを即座に無効化できます。また、停止フォームに時限停止のためのオプション時間数を入力できるようになりました（空欄 = 永続）。時限停止は指定した時間に自動的に解除されます。
- **サイトアナウンスバナー。** 管理者は **管理 → アナウンス** でサイト全体のアナウンスを記述し、表示を切り替えられます。有効にすると、すべてのページ上部（ナビゲーションヘッダーの下）に琥珀色のバナーでメッセージが表示されます。
- **管理者：メール送信。** 管理者は **管理 → メール送信** から、特定のユーザーアドレス、ロール内の全ユーザー、またはグループの全メンバーにプレーンテキストメールを送信できます。SMTP の設定が必要で、メールが無効な場合はページに通知が表示されます。
- **ログインロックアウトエラーメッセージ。** ログインページに、資格情報が無効、アカウントがロック中（残り分数付き）、アカウント停止、メール未確認 —— という具体的で翻訳済みのエラーメッセージが表示されるようになりました。以前の単一の「メールまたはパスワードが無効です」メッセージに代わるものです。4 種類のメッセージすべてが英語、繁體中文、简体中文、日本語に翻訳されています。

## 2026-06-26

### 追加
- **ノート表示での添付ファイル表示。** 画像またはファイルをアップロードした後、ノートページにノート本文下部の専用 **添付ファイル** セクションが表示されるようになりました：
  - **すべてのファイル**（画像、`.txt`、`.docx`、`.pdf`、動画など）に `📎 ファイル名（サイズ）` のダウンロードリンクが表示されます。
  - **画像** はダウンロードリンクの下にインラインプレビューも表示されます。
  - このセクションは `attachments` データベーステーブルに基づいており、ノート本文に OCR 抽出テキストのみが含まれていて Markdown に画像 URL が埋め込まれていない場合でも動作します。
- **ノートフォームのワンタイム送信ボタン。** 新規ノート / ノート編集ページのすべての送信ボタン（URL キャプチャ、画像アップロード、ファイルアップロード、手動ノートフォーム）が初回クリック後に即座に無効化されます。サーバーリダイレクトが届く前にダブルタップや低速ネットワークによって重複ノートが作成されるのを防ぎます。

### 修正
- **キャプチャクッキーのリプレイ攻撃対策。** 画像キャプチャ認証クッキーが、チェックのたびに（合格・不合格を問わず）即座に失効するようになりました。これにより、同じチャレンジトークンを再利用することができなくなります。次回ページ描画時には通常通り新しいクッキーと画像が発行されます。
- **メール再送信フォームにキャプチャチェックがなかった問題。** サインインページの「確認メールを再送」リンクが、サインイン / 登録 / パスワードを忘れた場合のフォームとは異なり、キャプチャを確認せずに新しい確認メールを送信していました。他の認証フローと一致するよう、最初にキャプチャを確認するようになりました。
- **OCR と画像解析がユーザーの選択した言語で出力されるようになりました。** 画像から抽出されたテキスト（OCR）と画像説明の出力が、ユーザーが選択した **要約言語**（例：繁體中文 / 简体中文）で生成されるようになりました。タイトル、要約、カテゴリ生成の既存の動作と一致します。Ollama とクラウド（OpenAI 互換）プロバイダーの両方が更新されており、言語指示は画像とともに送信されるプロンプトに追加されます。
- **OCR が重複コンテンツを生成していた問題（`[Image] ---` で区切られた同一テキストが 2 回）。** OCR がすでに ≥ 50 ルーンのテキストを返した場合、画像説明（`Describe`）の呼び出しがスキップされるようになりました。テキストが多いドキュメント画像では、OCR と Describe が同じビジョンモデルを使用して視覚的なコンテキストを追加するのではなくテキストを再読み取りし、`[Image]` プレフィックス付きの逐語的な複製を作成していました。OCR が何も返さないか非常に少ない写真や図では、Describe は引き続き実行されます。
- **`update.ps1`：サーバー出力がサイレントに破棄されていた問題。** `Start-Process` は stdout/stderr のリダイレクトなしで実行ファイルを起動していたため、起動エラーが見えなくなっていました。スクリプトは `cmd /c ... >> note-aura.log 2>&1` を介して起動するようになり（`update.sh` の `nohup >>` の動作に合わせる）、すべての出力をプロジェクトフォルダーの `note-aura.log` に追記します。

## 2026-06-21

### 追加
- **サインイン / サインアップ画像キャプチャ。** ログイン、登録、パスワードを忘れた場合のフォームに、歪んだ **画像認証コード**（5 文字の英数字）が表示されるようになりました。表示された文字を入力します。自動サインアップとパスワード推測を防ぐためのものです。外部サービス（Cloudflare/Google）も API キーも不要な組み込み機能なので、オフライン/イントラネット環境でも動作します。画像はサーバー側で生成され、コードは短命の HMAC 署名クッキーにのみ保持されます（ブラウザーに平文で送信されることはありません）。間違えると必ず新しい画像が表示されます。入力は大文字小文字を区別しません。ラベルは英語、繁體中文、简体中文、日本語に翻訳されています。
- **モバイルフィルター。** スマートフォンでは、ノートリストのメニュー直下に折りたたみ可能な **🔍 フィルター** パネルが表示されるようになりました。以前は大画面でのみ表示されていたカテゴリ（「プロジェクト」）とタグフィルターが利用できます。既存のフィルターデータを再利用し、アクティブなカテゴリ / タグをハイライトします。デスクトップのサイドバーは変更されていません。

## 2026-06-17

### 追加
- **Facebook リンクキャプチャ。** Facebook リンク（投稿、動画、リール、Watch ページ）を貼り付けると、Note-Aura は通常のウェブリンクや YouTube リンクと同様にコンテンツをキャプチャします —— タイトル、テキスト、利用可能なトランスクリプトを取得し、AI がタイトル、カテゴリ、タグ、要約を生成します。ソースバッジには `facebook` と表示されます。次の 3 つのフェッチパスが順番に試行されます：cookies 付き yt-dlp（最も豊富 —— 動画メタデータ + トランスクリプト）、認証済み headless Chrome（テキスト投稿）、公開 Open-Graph プレビュー（フォールバック、cookies 不要）。管理者は **管理 → AI 設定 → Facebook cookies (Netscape cookies.txt)** を通じて cookies パスを設定できます。空白のままでも、ほとんどのリンクで公開プレビューが取得できます。
- **AI 処理の停止ボタン。** ノートが `processing` 状態にある間、オーナーと編集者はステータスピルの隣に **停止** ボタンが表示されます。クリックすると進行中の AI ジョブがキャンセルされ、キューに入っているジョブが削除され、ノートが中立の「停止済み」状態に移行します。ノートビューには赤いエラーボックスの代わりに中立なボックス（「AI 処理が停止されました。」）と藍色の **再試行** ボタンが表示されます。再試行すると停止フラグがクリアされ、AI が再キューイングされます。すべての UI 文字列は英語、繁體中文、简体中文、日本語に翻訳されています。

### 追加
- **ワーカーライフサイクルロギング。** ワーカーはジョブ開始時に `processing note N job M (source=…)` を記録し、完了時に `completed note N job M` を記録するようになりました（以前は失敗時のみ記録）。これにより、遅い OCR が実際に実行されているのかどうかが判断できるようになります。
- **起動時の有効な AI 設定のログ出力。** Note-Aura 起動時に `AI config: ollama_url=… vision_model=… per-call timeout=…` がログ出力されるようになりました。OCR/タイムアウトの問題をログから診断できます —— たとえば、再デプロイ後に `ai.timeout_seconds` の値の変更が実際に反映されているか確認できます。
- **`update.sh` —— Linux/macOS アップデートヘルパー。** `update.ps1` の対応版：ソースディレクトリからサーバー上で実行し、すべてのデータを保持しながら停止 → 再ビルド → 再起動します。systemd を自動検出（`note-aura` ユニットを再起動、`NOTE_AURA_SERVICE` で上書き可能）またはバックグラウンドで再起動します。README および INSTALL.md §9 に記載されています。

### 修正
- **OCR がタイムアウト（「context deadline exceeded」）し、タイムアウト設定が無視されていた問題。** 両方の AI プロバイダーが 180 秒の HTTP クライアントタイムアウトをハードコードしており、config.yaml の `ai.timeout_seconds` を上げても Ollama/クラウドの呼び出しに効果がありませんでした。生成はストリーミングなしのため、その単一のタイムアウトが*応答全体*をカバーする必要があり、CPU またはリモートホストでの `deepseek-ocr` のコールドロード + 推論は通常 180 秒を超えるため、すべての OCR が失敗していました。プロバイダーの HTTP タイムアウトは `ai.timeout_seconds` によって制御されるようになり、デフォルト値は **600 秒** に引き上げられ、config のコメントに説明が追加されました。非常に遅いビジョンホストの場合は、さらに上げてください（例：900）。
- **YouTube のトランスクリプトが欠落することが多かった問題（説明文のみキャプチャされていた）。** トランスクリプトの取得は `yt-dlp … -o -` を使用していましたが、yt-dlp はサブタイトルを stdout にパイプ**しません** —— リテラルで `-.<lang>.vtt` というファイル名のファイルに書き込むため、プライマリキャプチャは常に空で返り（作業ディレクトリを散らかし）、すべての動画がレート制限を受けやすいフォールバックに頼っていました。yt-dlp は一時ディレクトリにキャプショントラックをダウンロードして `.vtt` ファイルを読み取るようになり、`--retries`/`--retry-sleep` で YouTube の HTTP 429 レート制限を緩和します。結果：字幕がある動画は確実にトランスクリプトをキャプチャできるようになりました。一時ディレクトリは自動削除されます（不要な `-.en.vtt` も残りません）。
- **`deepseek-ocr` での OCR / 画像解析が何も出力しなかった問題。** デフォルトのビジョンモデル `deepseek-ocr` は、特定のプロンプト向けにトレーニングされた専門家で改行に敏感なモデルです。Note-Aura は訓練されていない汎用的な指示（「この画像からすべてのテキストを抽出してください…」）を送信しており、空の / ガベージな出力が生成されていました。デフォルトの OCR と画像解析プロンプトが、モデルのドキュメント形式（`Free OCR.` / `Describe this image in detail.`、必要なリーディング改行付き）を使用するようになり、動作する OmniScribe リファレンスと一致します。以前はサイレントに握りつぶされていた画像解析の失敗が、管理者のシステムログに記録されるようになりました。**注意：** 管理 → AI 設定でカスタム OCR/画像プロンプトを設定している場合は、それをクリア（新しいデフォルトを取得するため）するか `Free OCR.` に設定してください。deepseek-ocr は Ollama ≥ v0.13.0 が必要です。
- **YouTube の anti-bot ページがトランスクリプトとしてキャプチャされていた問題。** YouTube/Google がサーバーの IP をレート制限した場合、キャプションエンドポイントは「We're sorry…」の HTML ページを返しましたが、本物のサブタイトルかどうかをチェックするものが何もなかったため、そのテキストがクリーニングされて動画のトランスクリプトとして保存されていました。キャプションペイロードは `WEBVTT` シグネチャがある場合にのみ受け入れられるようになりました（新しい `parseCaption` ガード。yt-dlp stdout パスとメタデータキャプション URL パスの両方に適用。後者はボット対策 HTML エクストラクター経由ではなくトラックを直接フェッチするようになりました）。ブロックされたフェッチは、ノートを汚染する代わりに動画の説明にフォールバックします。
- **停止済みノートの復活レース（ワーカー）。** `provider.*` AI 呼び出しが `Cancel()` と次の DB 書き込みの間の狭いウィンドウ内で完了した場合、その書き込みが停止ハンドラーの `status=failed, stopped=1` 状態を上書きし、ノートを不整合な `status=ready, stopped=1` 状態（通常の ready ノートとして表示される）のままにしていました。`run()` 内で停止/失敗状態を上書きする可能性のある各永続化書き込みの直前に、キャンセルガード（`ctx.Err()` チェック）が適用されるようになりました：`ApplyAIResult` 呼び出し（AI なしパス、早期保存、要約後）、`SetNoteTags`、`SetNoteCategory`、`ReplaceChunks`。キャンセルされたコンテキストはそれらの書き込みの前にエラーを返し、`process` が既存の no-op パスを取り、停止状態を保持できるようになります。
- **完全な多言語 UI。** 以前は英語のみだったパスワードリセット、パスワードを忘れた場合、Ask your notes、カレンダーページを含む、すべてのページが English / 繁體中文 / 简体中文 / 日本語 でレンダリングされるようになりました。`TestLanguageKeyParity` テストが、すべての翻訳キーが 4 言語すべてに存在することを強制します。
- **アカウントごとの言語記憶。** ユーザーが選択した UI 言語がアカウントに保存され、デバイス間で引き継がれます（ログイン前はクッキーでデバイスに記憶されます）。
- **ローカライズされたトランザクションメール。** 確認メールとパスワードリセットメールは受信者の言語で送信されます。招待メールは招待フォームで招待者が選んだ言語で送信されます（再送信時に再利用）。

### 変更
