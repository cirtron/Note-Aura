# 변경 이력

Note-Aura의 모든 주요 변경 사항은 여기에 최신 순으로 기록됩니다.

## 2026-06-27

### 추가
- **관리자: IP 주소 차단.** 관리자는 새로운 **관리 → 차단된 IP** 페이지에서 IP 주소를 차단할 수 있습니다. 차단된 IP는 세션이 확인되기 전 미들웨어 수준에서 HTTP 403으로 거부됩니다. 차단 목록은 선택적 사유 메모를 지원하며 각 IP가 차단된 시간을 표시합니다. 차단 해제는 즉시 적용됩니다.
- **관리자: 로그인 잠금 설정.** 설정된 횟수의 연속 로그인 실패 후 계정이 설정된 분 동안 잠깁니다. 두 임계값은 모두 **관리 → 로그인 잠금**에서 설정합니다(`login_max_attempts`, 기본값 5; `login_lockout_minutes`, 기본값 15). 로그인에 성공하면 카운터가 초기화됩니다. 관리자는 사용자 페이지에서 모든 사용자의 잠금을 해제할 수 있습니다.
- **관리자: 강제 로그아웃 및 시한부 정지.** 사용자 페이지에 **강제 로그아웃** 버튼이 추가되어 사용자의 모든 활성 세션을 즉시 무효화할 수 있습니다. 또한 정지 양식에서 시한부 정지를 위한 선택적 시간 수를 입력할 수 있습니다(공란 = 영구). 시한부 정지는 지정된 시간에 자동으로 해제됩니다.
- **사이트 공지 배너.** 관리자는 **관리 → 공지**에서 사이트 전체 공지를 작성하고 표시 여부를 전환할 수 있습니다. 활성화하면 메시지가 모든 페이지 상단(탐색 헤더 아래)의 황색 배너에 표시됩니다.
- **관리자: 이메일 발송.** 관리자는 **관리 → 이메일 발송**을 통해 특정 사용자, 특정 역할의 모든 사용자, 또는 그룹의 모든 구성원에게 일반 텍스트 이메일을 보낼 수 있습니다. SMTP 설정이 필요하며, 이메일이 비활성화된 경우 페이지에 알림이 표시됩니다.
- **로그인 잠금 오류 메시지.** 로그인 페이지에 잘못된 자격 증명, 계정 잠김(남은 분 표시), 계정 정지, 이메일 미인증에 대한 구체적이고 번역된 오류 메시지가 표시됩니다. 기존의 단일한 "잘못된 이메일 또는 비밀번호" 메시지를 대체합니다. 네 가지 메시지 모두 영어, 繁體中文, 简体中文, 日本語로 번역되었습니다.

## 2026-06-26

### 추가
- **노트 보기에서 첨부 파일 표시.** 이미지나 파일을 업로드한 후, 노트 페이지에 노트 본문 하단에 전용 **첨부 파일** 섹션이 표시됩니다:
  - **모든 파일**(이미지, `.txt`, `.docx`, `.pdf`, 동영상 등)에 `📎 파일명(크기)` 다운로드 링크가 표시됩니다.
  - **이미지**는 다운로드 링크 아래에 인라인 미리보기도 표시됩니다.
  - 이 섹션은 `attachments` 데이터베이스 테이블을 기반으로 하므로, 노트 본문에 OCR 추출 텍스트만 있고 Markdown에 이미지 URL이 포함되지 않은 경우에도 동작합니다.
- **노트 양식의 일회성 제출 버튼.** 새 노트 / 노트 편집 페이지의 모든 제출 버튼(URL 캡처, 이미지 업로드, 파일 업로드, 수동 노트 양식)이 첫 번째 클릭 후 즉시 비활성화됩니다. 서버 리디렉션이 도착하기 전에 더블 탭이나 느린 네트워크로 인해 중복 노트가 생성되는 것을 방지합니다.

### 수정
- **캡차 쿠키 재사용 공격 방지.** 이미지 캡차 인증 쿠키가 이제 모든 확인 후(통과 또는 실패 여부에 관계없이) 즉시 만료되어, 동일한 챌린지 토큰을 재사용할 수 없게 됩니다. 다음 페이지 렌더링 시 새로운 쿠키와 이미지가 발급됩니다.
- **이메일 재발송 양식에 캡차 확인이 없었던 문제.** 로그인 페이지의 "인증 이메일 재발송" 링크가 로그인 / 회원가입 / 비밀번호 찾기 양식과 달리 캡차를 확인하지 않고 새 인증 이메일을 보내고 있었습니다. 이제 나머지 인증 흐름과 일치하도록 먼저 캡차를 확인합니다.
- **OCR 및 이미지 분석이 사용자의 선택 언어로 출력됩니다.** 이미지에서 추출된 텍스트(OCR)와 이미지 설명 출력이 이제 사용자가 선택한 **요약 언어**(예: 繁體中文 / 简体中文)로 생성됩니다. 기존 제목, 요약, 카테고리 생성 동작과 일치합니다. Ollama와 클라우드(OpenAI 호환) 제공자 모두 업데이트되었으며, 언어 지시문이 이미지와 함께 전송되는 프롬프트에 추가됩니다.
- **OCR이 중복 콘텐츠를 생성했던 문제(`[Image] ---`로 구분된 동일 텍스트가 두 번).** OCR이 이미 ≥ 50 룬의 텍스트를 반환한 경우 이미지 설명(`Describe`) 호출이 건너뛰어집니다. 텍스트가 많은 문서 이미지의 경우, OCR과 Describe가 동일한 비전 모델을 사용하여 시각적 컨텍스트를 추가하는 대신 텍스트를 다시 읽어 `[Image]` 접두사가 붙은 축자적 중복을 만들었습니다. OCR이 아무것도 반환하지 않거나 매우 적게 반환하는 사진과 다이어그램에서는 Describe가 계속 실행됩니다.
- **`update.ps1`: 서버 출력이 자동으로 삭제되었던 문제.** `Start-Process`가 stdout/stderr 리디렉션 없이 실행 파일을 실행하여 시작 오류가 보이지 않았습니다. 스크립트는 이제 `cmd /c ... >> note-aura.log 2>&1`을 통해 실행되며(`update.sh`의 `nohup >>` 동작과 일치), 모든 출력을 프로젝트 폴더의 `note-aura.log`에 추가합니다.

## 2026-06-21

### 추가
- **로그인 / 회원가입 이미지 캡차.** 로그인, 회원가입, 비밀번호 찾기 양식에 왜곡된 **이미지 인증 코드**(5자의 영숫자)가 표시됩니다. 표시된 문자를 입력하면 됩니다. 자동화된 회원가입과 비밀번호 추측을 방지하며, 외부 서비스(Cloudflare/Google)나 API 키가 필요 없는 내장 기능이므로 오프라인/인트라넷 환경에서도 작동합니다. 이미지는 서버에서 생성되며 코드는 단기 HMAC 서명 쿠키에만 저장됩니다(브라우저에 평문으로 전송되지 않음). 틀릴 경우 항상 새 이미지가 표시됩니다. 입력은 대소문자를 구분하지 않습니다. 레이블은 영어, 繁體中文, 简体中文, 日本語로 번역되었습니다.
- **모바일 필터.** 스마트폰에서 노트 목록의 메뉴 바로 아래에 접을 수 있는 **🔍 필터** 패널이 표시됩니다. 이전에는 큰 화면에서만 볼 수 있었던 카테고리("프로젝트")와 태그 필터를 제공합니다. 기존 필터 데이터를 재사용하며 활성 카테고리/태그를 강조 표시합니다. 데스크톱 사이드바는 변경되지 않았습니다.

## 2026-06-17

### 추가
- **Facebook 링크 캡처.** Facebook 링크(게시물, 동영상, 릴스, Watch 페이지)를 붙여넣으면 Note-Aura가 일반 웹 링크나 YouTube 링크처럼 콘텐츠를 캡처합니다 —— 제목, 텍스트, 이용 가능한 자막을 가져온 후 AI가 제목, 카테고리, 태그, 요약을 생성합니다. 소스 뱃지는 `facebook`을 표시합니다. 다음 세 가지 가져오기 경로가 순서대로 시도됩니다: cookies가 포함된 yt-dlp(가장 풍부함 —— 동영상 메타데이터 + 자막), 인증된 headless Chrome(텍스트 게시물), 공개 Open-Graph 미리보기(폴백, cookies 불필요). 관리자는 **관리 → AI 설정 → Facebook cookies (Netscape cookies.txt)**를 통해 cookies 경로를 설정할 수 있습니다. 비워두어도 대부분의 링크에서 공개 미리보기를 얻을 수 있습니다.
- **AI 처리 중지 버튼.** 노트가 `processing` 상태에 있는 동안 소유자와 편집자는 상태 필 옆에 **중지** 버튼이 표시됩니다. 클릭하면 진행 중인 AI 작업이 취소되고, 대기 중인 작업이 삭제되며, 노트가 중립적인 "중지됨" 상태로 전환됩니다. 노트 보기에 빨간색 오류 상자 대신 중립적인 상자("AI 처리가 중지되었습니다.")와 남색 **재시도** 버튼이 표시됩니다. 재시도하면 중지 플래그가 지워지고 AI가 재대기열에 추가됩니다. 모든 UI 문자열은 영어, 繁體中文, 简体中文, 日本語로 번역되었습니다.

### 추가
- **워커 생명주기 로깅.** 워커가 이제 작업 시작 시 `processing note N job M (source=…)`을, 완료 시 `completed note N job M`을 로그에 기록합니다(이전에는 실패 시에만 기록). 느린 OCR이 실제로 실행 중인지 아니면 전혀 처리되지 않는지 파악할 수 있게 됩니다.
- **시작 시 유효한 AI 설정 로그 출력.** Note-Aura 시작 시 `AI config: ollama_url=… vision_model=… per-call timeout=…`이 로그에 기록됩니다. OCR/타임아웃 문제를 로그에서 진단할 수 있습니다 —— 예를 들어 재배포 후 `ai.timeout_seconds` 값 변경이 실제로 적용되었는지 확인할 수 있습니다.
- **`update.sh` —— Linux/macOS 업데이트 도우미.** `update.ps1`의 대응 버전: 소스 디렉터리에서 서버 위에서 실행하면 모든 데이터를 보존하면서 중지 → 재빌드 → 재시작합니다. systemd를 자동 감지하거나(`note-aura` 유닛 재시작, `NOTE_AURA_SERVICE`로 재정의 가능) 백그라운드에서 재시작합니다. README 및 INSTALL.md §9에 문서화되어 있습니다.

### 수정
- **OCR이 타임아웃("context deadline exceeded")되고 타임아웃 설정이 무시되었던 문제.** 두 AI 제공자 모두 180초의 HTTP 클라이언트 타임아웃을 하드코딩했기 때문에 config.yaml의 `ai.timeout_seconds`를 올려도 Ollama/클라우드 호출에 효과가 없었습니다. 생성이 스트리밍이 아니므로 단일 타임아웃이 *전체* 응답을 커버해야 하는데, CPU 또는 원격 호스트에서 `deepseek-ocr` 콜드 로드 + 추론은 통상 180초를 초과하여 모든 OCR이 실패했습니다. 제공자의 HTTP 타임아웃이 이제 `ai.timeout_seconds`에 의해 제어되며, 기본값이 **600초**로 올라갔고, config 주석에 설명이 추가되었습니다. 비전 호스트가 매우 느린 경우 더 올리세요(예: 900).
- **YouTube 자막이 자주 누락되었던 문제(설명만 캡처됨).** 자막 가져오기에 `yt-dlp … -o -`를 사용했지만 yt-dlp는 자막을 stdout으로 파이프하지 **않습니다** —— 리터럴로 `-.<lang>.vtt`라는 이름의 파일에 기록하여, 기본 캡처가 항상 빈 상태로 반환되고(작업 디렉터리에 파일을 남기며) 모든 동영상이 속도 제한을 받기 쉬운 폴백에 의존했습니다. yt-dlp가 이제 임시 디렉터리에 캡션 트랙을 다운로드하고 `.vtt` 파일을 읽으며, `--retries`/`--retry-sleep`으로 YouTube의 HTTP 429 속도 제한을 완화합니다. 결과: 자막이 있는 동영상은 이제 자막을 안정적으로 캡처합니다. 임시 디렉터리는 자동으로 제거됩니다(더 이상 불필요한 `-.en.vtt` 파일이 남지 않습니다).
- **`deepseek-ocr`에서 OCR / 이미지 분석이 아무것도 출력하지 않았던 문제.** 기본 비전 모델 `deepseek-ocr`은 특정 프롬프트에 맞게 훈련된 전문가형, 줄 바꿈에 민감한 모델입니다. Note-Aura가 훈련되지 않은 일반 지시("이 이미지에서 모든 텍스트를 추출하세요…")를 보내어 빈/가비지 출력이 생성되었습니다. 기본 OCR 및 이미지 분석 프롬프트가 이제 모델의 문서 형식(`Free OCR.` / `Describe this image in detail.`, 필요한 선행 줄 바꿈 포함)을 사용하며, 동작하는 OmniScribe 참조와 일치합니다. 이전에 자동으로 무시되었던 이미지 분석 실패가 이제 관리자 시스템 로그에 기록됩니다. **참고:** 관리 → AI 설정에서 사용자 정의 OCR/이미지 프롬프트를 설정한 경우 지워서(새 기본값을 가져오기 위해) `Free OCR.`으로 설정하세요. deepseek-ocr에는 Ollama ≥ v0.13.0이 필요합니다.
- **YouTube 봇 방지 페이지가 자막으로 캡처되었던 문제.** YouTube/Google이 서버 IP를 속도 제한할 때 자막 엔드포인트가 "We're sorry…" HTML 페이지를 반환했지만 실제 자막인지 확인하는 것이 없었기 때문에 해당 텍스트가 정제되어 동영상의 자막으로 저장되었습니다. 캡션 페이로드가 `WEBVTT` 서명을 가진 경우에만 허용됩니다(새 `parseCaption` 가드, yt-dlp stdout 경로와 메타데이터 캡션 URL 경로 모두에 적용됨; 후자는 이제 봇 허용 HTML 추출기가 아닌 트랙을 직접 가져옵니다). 차단된 가져오기는 노트를 오염시키는 대신 동영상 설명으로 폴백합니다.
- **중지된 노트의 부활 경쟁 상태(워커).** `provider.*` AI 호출이 `Cancel()`과 다음 DB 쓰기 사이의 좁은 창 내에서 완료된 경우, 그 쓰기가 중지 핸들러의 `status=failed, stopped=1` 상태를 덮어써서 노트를 불일치한 `status=ready, stopped=1` 상태(일반 ready 노트로 표시됨)로 남겨두었습니다. `run()` 내에서 중지/실패 상태를 덮어쓸 수 있는 각 영속성 쓰기 직전에 취소 가드(`ctx.Err()` 확인)가 적용됩니다: `ApplyAIResult` 호출(AI 없음 경로, 조기 저장, 요약 후), `SetNoteTags`, `SetNoteCategory`, `ReplaceChunks`. 취소된 컨텍스트는 해당 쓰기 전에 오류를 반환하여 `process`가 기존 no-op 경로를 따르고 중지 상태를 보존할 수 있게 합니다.
- **완전한 다국어 UI.** 이전에는 영어만 지원했던 비밀번호 재설정, 비밀번호 찾기, Ask your notes, 캘린더 페이지를 포함한 모든 페이지가 이제 English / 繁體中文 / 简体中文 / 日本語 로 렌더링됩니다. `TestLanguageKeyParity` 테스트가 모든 번역 키가 4개 언어 모두에 존재하도록 강제합니다.
- **계정별 언어 기억.** 사용자가 선택한 UI 언어가 계정에 저장되어 모든 장치에서 유지됩니다(로그인 전에는 쿠키를 통해 장치에 기억됩니다).
- **현지화된 트랜잭션 이메일.** 인증 및 비밀번호 재설정 이메일이 수신자의 언어로 발송됩니다. 초대 이메일은 초대 양식에서 초대자가 선택한 언어로 발송됩니다(재발송 시 재사용됨).
