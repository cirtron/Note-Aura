# 更新日志

Note-Aura 所有重要变更均记录于此，最新在前。

## 2026-06-27

### 新增
- **管理员：IP 地址封锁。** 管理员现在可从全新的 **管理 → 封锁的 IP 地址** 页封锁 IP 地址。被封锁的 IP 会在中间件解析会话之前以 HTTP 403 拒绝。封锁列表支持选填的原因说明，并显示每个 IP 被封锁的时间。解除封锁即时生效。
- **管理员：可配置的登录锁定。** 连续登录失败达设置次数后，账号会锁定指定的分钟数。两个阈值均在 **管理 → 登录锁定** 中设置（`login_max_attempts`，默认 5；`login_lockout_minutes`，默认 15）。成功登录后计数器重置。管理员可在用户页面解除任何用户的锁定。
- **管理员：强制登出与定时暂停。** 用户页面新增 **强制登出** 按钮，可立即让某用户所有进行中的会话失效；暂停表单也支持输入选填的小时数进行定时暂停（留空 = 永久）。定时暂停将在指定时间自动解除。
- **网站公告横幅。** 管理员可在 **管理 → 网站公告** 撰写全站公告并切换显示。启用后，消息会以琥珀色横幅显示在每个页面顶部（导航栏下方）。
- **管理员：发送邮件。** 管理员可通过 **管理 → 发送邮件** 发送纯文本邮件给指定用户、某角色的所有用户，或某群组的所有成员。需要配置 SMTP；未配置时页面会显示提示。
- **登录锁定错误消息。** 登录页现在会针对以下情况显示具体且已翻译的错误消息：凭证无效、账号锁定（含剩余分钟数）、账号暂停、邮件未验证 —— 不再只显示单一的"邮件或密码无效"。四种消息均已翻译成英文、繁體中文、简体中文与日本語。

## 2026-06-26

### 新增
- **笔记浏览页显示附件。** 上传图片或文件后，笔记页面现在会在正文底部显示专属的 **附件** 区块：
  - **每个文件**（图片、`.txt`、`.docx`、`.pdf`、视频等）均有 `📎 文件名（大小）` 下载链接。
  - **图片** 还会在下载链接下方显示内嵌预览。
  - 此区块以 `attachments` 数据库表为依据，即使笔记正文只有 OCR 提取的文字而 Markdown 中未嵌入图片 URL 也同样有效。
- **笔记表单的一次性提交按钮。** 新增笔记 / 编辑笔记页面上所有提交按钮（URL 捕获、图片上传、文件上传、手动笔记表单）在第一次点击后立即禁用，防止在服务器重定向前因双击或网络缓慢而创建重复笔记。

### 修复
- **验证码 Cookie 重放攻击防护。** 图片验证码验证 cookie 现在在每次验证后（无论通过或失败）立即过期，因此同一个挑战 token 无法被重复使用。下次页面渲染时会如常发放新的 cookie 与图片。
- **重新发送验证邮件表单缺少验证码检查。** 登录页上的"重新发送验证邮件"链接在发送新验证邮件之前没有验证验证码，与登录 / 注册 / 忘记密码表单不一致。现在统一先验证验证码，与其他验证流程一致。
- **OCR 与图片分析现在以用户选择的语言输出。** 从图片提取的文字（OCR）与图片描述输出现在使用用户选择的 **摘要语言** 生成（例如繁體中文 / 简体中文）。这与现有的标题、摘要、分类生成行为一致。Ollama 与云端（OpenAI 兼容）提供者均已更新；语言指示附加在发送图片的提示词中。
- **OCR 产生重复内容（相同文字出现两次，以 `[Image] ---` 分隔）。** 当 OCR 已返回 ≥ 50 个 rune 的文字时，现在会跳过图片描述（`Describe`）调用。对于文字密集的文档图片，OCR 与 Describe 使用同一个视觉模型，会重新读取文字而非提供视觉背景，造成带有 `[Image]` 前缀的逐字重复。对于 OCR 返回空白或很少文字的照片与图表，Describe 仍照常执行。
- **`update.ps1`：服务器输出被静默丢弃。** `Start-Process` 启动可执行文件时未重定向 stdout/stderr，因此启动错误不可见。脚本现在通过 `cmd /c ... >> note-aura.log 2>&1` 启动（与 `update.sh` 的 `nohup >>` 行为一致），将所有输出追加到项目文件夹的 `note-aura.log`。

## 2026-06-21

### 新增
- **登录 / 注册图片验证码。** 登录、注册、忘记密码表单现在显示扭曲的 **图片验证码**（5 个英数字符）；你需要输入图片中显示的字符。此功能可阻止自动化注册与密码猜测攻击，为内置功能，不需要外部服务（Cloudflare/Google）也不需要 API 密钥，因此适用于离线/企业内网部署。图片在服务器生成，验证码只存放在短期有效的 HMAC 签署 cookie 中（不以明文传送到浏览器）；输入错误时一定会换一张新图片。输入不区分大小写。标签已翻译成英文、繁體中文、简体中文与日本語。
- **移动端筛选器。** 在手机上，笔记列表现在有一个可折叠的 **🔍 筛选** 面板，直接位于菜单下方，提供先前只在较大屏幕上可见的分类（"项目"）与标签筛选器。此面板复用现有的筛选数据，并高亮当前使用中的分类/标签；桌面版侧边栏不受影响。

## 2026-06-17

### 新增
- **Facebook 链接捕获。** 粘贴 Facebook 链接（帖子、视频、Reels 或 Watch 页面），Note-Aura 会像处理普通网页或 YouTube 链接一样捕获其内容 —— 提取标题、文字与任何可用的字幕，然后 AI 生成标题、分类、标签与摘要。来源标签显示 `facebook`。会依次尝试三种捕获方式：yt-dlp 含 cookies（最丰富 —— 视频元数据 + 字幕）、已验证 headless Chrome（文字帖子）、公开 Open-Graph 预览（备用，不需要 cookies）。管理员可通过 **管理 → AI 设置 → Facebook cookies (Netscape cookies.txt)** 配置 cookies 路径；留空时仍可获取大多数链接的公开预览。
- **AI 处理的停止按钮。** 笔记在 `processing` 状态时，所有者和编辑者会在状态标签旁看到 **停止** 按钮。点击后会取消进行中的 AI 任务、删除排队中的任务，并将笔记转换至中性的"已停止"状态。笔记浏览页会显示中性框（"AI 处理已停止。"）与靛蓝色 **重试** 按钮，取代原本的红色错误框。重试时会清除已停止标志并重新排队 AI。所有 UI 字符串均已翻译成英文、繁體中文、简体中文与日本語。

### 新增
- **Worker 生命周期日志。** Worker 现在在开始任务时记录 `processing note N job M (source=…)`，完成时记录 `completed note N job M`（之前只在失败时记录）。这使得我们能够看出缓慢的 OCR 是否真的在执行，还是根本没被处理。
- **启动时记录有效的 AI 设置。** Note-Aura 启动时现在记录 `AI config: ollama_url=… vision_model=… per-call timeout=…`，这样 OCR/超时问题可以从日志诊断 —— 例如确认在重新部署后调高 `ai.timeout_seconds` 是否真的生效。
- **`update.sh` —— Linux/macOS 更新工具。** `update.ps1` 的对应版本：在服务器上从源码目录执行，可停止 → 重新编译 → 重新启动，同时保留所有数据。自动检测 systemd（重新启动 `note-aura` 服务，可用 `NOTE_AURA_SERVICE` 覆盖）或在后台重新启动。已在 README 与 INSTALL.md §9 中记录。

### 修复
- **OCR 超时（"context deadline exceeded"）且超时设置被忽略。** 两个 AI 提供者都硬编码了 180 秒的 HTTP 客户端超时，因此调高 config.yaml 中的 `ai.timeout_seconds` 对 Ollama/云端调用没有效果。由于生成是非流式的，那个单一超时必须涵盖*整个*响应 —— 而在 CPU 或远程主机上冷启动 `deepseek-ocr` 并推理的时间通常超过 180 秒，导致每次 OCR 失败。提供者的 HTTP 超时现在由 `ai.timeout_seconds` 控制，默认值已提高至 **600 秒**，且 config 说明已解释此设置。如果你的视觉主机非常慢，可进一步调高（例如 900）。
- **YouTube 字幕经常缺失（只捕获到描述）。** 字幕捕获使用 `yt-dlp … -o -`，但 yt-dlp **不会**将字幕输出到 stdout —— 它会写入一个字面上叫做 `-.<lang>.vtt` 的文件，因此主要捕获总是返回空白（并在工作目录留下垃圾文件），导致每个视频都退回到容易受速率限制的备用方案。yt-dlp 现在将字幕轨下载到临时目录并读取 `.vtt` 文件，并加入 `--retries`/`--retry-sleep` 来缓解 YouTube 的 HTTP 429 速率限制。结果：有字幕的视频现在能可靠地捕获字幕；临时目录自动移除（不再有多余的 `-.en.vtt`）。
- **`deepseek-ocr` 的 OCR / 图片分析没有产生任何输出。** 默认视觉模型 `deepseek-ocr` 是一个针对特定提示词训练的专家型、对换行符敏感的模型；Note-Aura 发送了它未被训练的通用指示（"从这张图片提取所有文字…"），导致空白/乱码输出。默认的 OCR 与图片分析提示词现在使用模型文档中的格式（`Free OCR.` / `Describe this image in detail.`，以及必要的前导换行符），与有效的 OmniScribe 参考一致。图片分析失败原本被静默吞掉，现在会记录到管理员系统日志。**注意：** 如果你在 管理 → AI 设置 下配置了自定义 OCR/图片提示词，请清除它（以获取新默认值）或将其设为 `Free OCR.`；deepseek-ocr 还需要 Ollama ≥ v0.13.0。
- **YouTube 反机器人页面被捕获为字幕。** 当 YouTube/Google 对服务器 IP 进行速率限制时，字幕端点返回了"We're sorry…"HTML 页面，但没有任何检查确认它是真正的字幕 —— 因此该文字被清理后存为视频字幕。字幕内容现在只有在带有 `WEBVTT` 签名时才被接受（新的 `parseCaption` 防护，同时应用于 yt-dlp stdout 路径和元数据字幕 URL 路径；后者现在直接获取轨道原始内容，而非通过防机器人 HTML 提取器）。被封锁的获取会退回视频描述，而不是污染笔记。
- **已停止笔记的复活竞争（worker）。** 如果某个 `provider.*` AI 调用恰好在 `Cancel()` 与下一次 DB 写入之间的窄窗口内完成，该写入会覆盖停止处理器的 `status=failed, stopped=1` 状态，使笔记留在不一致的 `status=ready, stopped=1` 状态（显示为正常的就绪笔记）。现在在 `run()` 中每个可能覆盖已停止/失败状态的持久化写入之前立即应用取消防护（`ctx.Err()` 检查）：`ApplyAIResult` 调用（无 AI 路径、早期保存、摘要后）、`SetNoteTags`、`SetNoteCategory` 和 `ReplaceChunks`。已取消的 context 现在会在这些写入之前返回错误，让 `process` 走现有的无操作路径，并保留已停止状态。
- **完整多语界面。** 每个页面现在均以 English / 繁體中文 / 简体中文 / 日本語 渲染，包括先前只有英文的密码重置、忘记密码、问笔记、日历等页面。`TestLanguageKeyParity` 测试强制确保每个翻译键在所有四种语言中都存在。
- **个人语言记忆。** 用户选择的界面语言保存在其账号中，并在各设备间跟随（登录前以 cookie 记忆）。
- **本地化的事务性邮件。** 验证和密码重置邮件以收件人的语言发送；邀请邮件以发件人在邀请表单上选择的语言发送（在"重新发送"时复用）。
