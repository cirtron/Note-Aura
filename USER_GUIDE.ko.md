# Note-Aura — 사용자 가이드

Note-Aura는 당신의 **AI 지식 받은편지함**입니다. 무엇이든 던져 넣으세요 —— 직접 작성한 메모,
웹 링크, YouTube 동영상, 또는 이미지 —— AI가 제목을 달고, 요약하고, 태그를 붙이고, 색인을
만듭니다. 나중에 키워드 검색이나 **노트에 질문하기**로 원하는 내용을 찾을 수 있습니다.

> **한눈에 보기:** 인라인 이미지가 포함된 Markdown 노트 · 웹/YouTube/
> 이미지(OCR)/**파일**/**이메일**에서 캡처 · AI **제목 + 요약 + 태그 + 카테고리**
>(빈 필드만 채움) · 키워드 검색,
> **정렬 및 페이지네이션**, "노트에 질문하기"(RAG) · **하위 카테고리**, 태그 및
> 이메일 알림이 있는 캘린더 · 공휴일 · **일괄 삭제** ·
> 노트 **가져오기/내보내기** · 개인 또는 **그룹**과 공유(공동 관리자 및
> 읽기/쓰기 제어) · 사용자 차단 · 새 사용자 초대 · 이메일 인증 ·
> 다국어 UI(브라우저 따름) · 나만의 AI 키 사용(프롬프트도). 관리자에게는
> **대시보드**가 있으며 사용자(정지/삭제/강제 로그아웃,
> **시한부 정지**, **로그인 잠금**), 역할 및 할당량, 허용된 업로드 유형,
> 브랜딩, AI 모델/프롬프트, 공휴일, 이메일, **HTTPS**, **IP 차단**,
> **사이트 공지**, 가입 공개 여부를 관리할 수 있습니다.

## 1. 시작하기

### 계정 만들기
1. Note-Aura를 열고 로그인 페이지에서 **계정 만들기**를 클릭합니다.
2. 이메일과 비밀번호(최소 6자)를 입력합니다.
3. 양식의 **인증 이미지**(캡차)에 표시된 문자를 입력합니다 ——
   자동화된 가입을 차단하기 위한 것입니다. 대소문자를 구분하지 않습니다.
4. 사이트에 이메일이 활성화된 경우 **인증 링크**를 받게 됩니다 —— 링크를 열어
   주소를 인증한 후 로그인합니다. (받지 못했나요? 로그인 페이지에서
   **인증 이메일 재발송**을 사용하세요.)
5. 다른 사용자에게 **초대**받은 경우, 등록 페이지에 이메일이 미리 입력되어 있으며
   자동으로 인증됩니다 —— 비밀번호만 설정하면 됩니다.

> 새 배포에서 처음 생성된 계정이 **관리자**가 됩니다.

### 로그인
이메일과 비밀번호를 입력하고 **인증 이미지**(캡차)의 문자를 입력합니다.
같은 캡차가 **비밀번호 찾기** 양식도 보호합니다.
틀리면 새 이미지가 나타납니다. 세션은 약 30일 동안 유지됩니다.

## 2. 인터페이스

상단 바에는 두 행이 있습니다:

- **첫 번째 행:** 로고, 메인 메뉴(**노트, 캘린더, 질문, 공유, 그룹,
  설정**, 관리자는 **관리**도 표시), **+ 노트 추가** 버튼.
- **두 번째 행(오른쪽):** **언어 선택기**, 이메일 주소, **로그아웃**.

### 언어
선택기에서 언어를 선택합니다(English / 繁體中文 / 简体中文 / 日本語). 선택하지 않으면
Note-Aura가 자동으로 **브라우저 언어**를 따릅니다. 선택은 **계정에 저장**되므로
로그인하는 모든 기기에서 유지됩니다. (로그인 전에는 쿠키를 통해 기기에 저장됩니다.)
Note-Aura에서 발송하는 이메일 —— 인증 및 비밀번호 재설정 —— 도 이 언어로 발송됩니다.

## 3. 노트 추가

**+ 노트 추가**를 클릭합니다. 캡처 방법은 세 가지입니다:

### 노트 작성(Markdown)
Markdown 편집기에서 입력합니다(굵게, 제목, 목록, 링크, 코드). 다음도 할 수 있습니다:
- **이미지 삽입** —— 이미지 버튼을 사용하거나 이미지를 편집기에 직접 **드래그 앤 드롭**하거나 **붙여넣기**합니다.
- 선택적으로 **제목**, **카테고리**, **태그**, **일정**(캘린더 참조), **요약 언어**를 추가합니다.

제목을 비워두면 AI가 대신 작성해 줍니다.

### 링크에서 추가(웹, YouTube 또는 Facebook)
URL을 붙여넣습니다. Note-Aura가 페이지 텍스트(또는 동영상 자막)를 가져온 후
AI가 노트로 요약합니다. 원하는 경우 **요약 언어**를 선택하세요.

**Facebook** 게시물, 동영상, 릴스는 동일하게 지원됩니다 —— URL을 붙여넣으면 Note-Aura가 접근 가능한 콘텐츠를 캡처합니다. 소스 배지에는 `facebook`이 표시됩니다. **설정 → Facebook Cookies**에서 자신의 쿠키를 붙여넣을 수 있으며, 관리자도 Admin 설정에서 공유 대체값을 설정할 수 있습니다. 쿠키가 없으면 공개 Open-Graph 미리보기(제목 + 설명)를 사용하며, 대부분의 공개 게시물에 적용됩니다.

#### Facebook Cookies 설정 (더 풍부한 콘텐츠 가져오기)

쿠키를 사용하면 Note-Aura가 당신으로서 Facebook에 접근하여 전체 게시물 텍스트, 동영상 자막, 로그인 후에만 볼 수 있는 콘텐츠를 가져올 수 있습니다.

1. 브라우저에서 **Facebook에 로그인**합니다.
2. 쿠키 내보내기 확장 프로그램을 설치합니다 — 예: **Get cookies.txt LOCALLY** (Chrome/Firefox 지원).
3. `facebook.com`에서 확장 프로그램을 열고 **Netscape 형식**으로 내보냅니다 (파일이 `# Netscape HTTP Cookie File`로 시작).
4. 내보낸 파일의 전체 텍스트 내용을 복사합니다.
5. Note-Aura에서 **설정 → Facebook Cookies**로 이동하여 텍스트를 붙여넣고 저장합니다.

개인 쿠키가 우선 적용되며, 관리자의 공유 쿠키는 설정하지 않은 사용자의 대체값으로 작동합니다.

> **쿠키 만료:** Facebook 세션 쿠키는 주기적으로 만료됩니다. 인증된 캡처가 작동하지 않으면 다시 내보내고 새 쿠키를 붙여넣으세요.

> **웹 링크 제한:** **로그인이 필요한 페이지**(Feishu, Google Docs, 접근 제어가 있는 Notion 등)는 가져올 수 없습니다 —— Note-Aura는 해당 서비스의 계정이 없습니다. 일부 **YouTube 동영상**(연령 제한 또는 지역 제한 또는 기타 이유)은 yt-dlp가 설치되어 있어도 이용할 수 없을 수 있습니다.

### 이미지 추가(OCR)
사진이나 스크린샷을 업로드합니다. AI가 텍스트를 추출(OCR)하고 요약합니다.
OCR 출력은 선택한 **요약 언어**로 작성됩니다(자동으로 설정하면 이미지 내용에서
자동 감지됩니다).

### 파일 업로드
역할에서 허용되는 경우 **파일 업로드** 옵션이 나타납니다(허용된 파일 유형도 표시됩니다).
각 유형의 처리 방법:
- **이미지** → OCR(위와 동일).
- **텍스트 파일**(`.txt`, `.md`, `.csv` 등) → 내용이 노트 본문이 됩니다.
- **기타 파일**(동영상, 오디오, PDF, Office 등) → 노트에 링크된 다운로드 가능한
  첨부 파일로 저장됩니다.

> 각 역할이 업로드할 수 있는 파일 유형은 관리자가 설정합니다.

### 노트에서 업로드된 파일 보기
파일을 업로드하면 노트 페이지의 노트 본문 하단에 **첨부 파일** 섹션이 표시됩니다:
- **모든 파일** —— 이미지, `.txt`, `.docx`, `.pdf`, 동영상 등 —— 에
  `📎 파일명(크기)` 다운로드 링크가 있습니다.
- **이미지**는 다운로드 링크 아래에 인라인 미리보기도 표시됩니다.

### 이메일로 추가
**이메일**로도 내용을 보낼 수 있습니다 —— [이메일 → 노트](#11-이메일--노트)를 참조하세요.

> **캡처는 즉시 이루어집니다.** 노트는 즉시 저장되며 AI가 백그라운드에서
> 처리하는 동안 *처리 중…* 으로 표시됩니다. 완료되면 *준비 완료*가 됩니다.
> AI를 일시적으로 사용할 수 없더라도 캡처된 콘텐츠는 보존됩니다.

## 4. AI 처리 작동 방식

내용을 캡처하면 AI가 자동으로 여러 작업을 수행합니다 —— 단 **비워둔 필드에만**
(직접 설정한 제목, 태그, 카테고리는 덮어쓰지 않습니다):
- **제목** —— 짧은 제목(직접 입력하지 않은 경우에만).
- **요약** —— 2~3문장 요약.
- **태그** —— 몇 가지 주제 태그.
- **카테고리** —— 적합한 기존 카테고리를 선택하거나 새 카테고리를 제안합니다.
- **색인** —— 검색하고 *노트에 질문*할 수 있도록 콘텐츠를 임베딩합니다.

### 요약 언어
모든 캡처 양식에서 **요약 언어**를 선택합니다:
- **자동(내용 일치)** —— 요약이 콘텐츠 언어를 따릅니다.
- 또는 특정 언어 선택 —— 제목, 요약, **OCR 텍스트**,
  **이미지 설명** 모두 해당 언어로 작성됩니다.

### AI 처리 중지
노트가 *처리 중*인 동안 상태 배지 옆에 **■ 중지** 버튼이 나타납니다.
클릭하면 진행 중인 AI 작업이 즉시 취소됩니다 —— 캡처된 콘텐츠는 보존되고
노트는 중립적인 **중지됨** 상태가 되며 "AI 처리가 중지되었습니다."라는 메시지와
남색 **↻ AI 처리 다시 시도** 버튼이 표시됩니다. 준비가 되면 다시 시도를 클릭하여
AI를 재실행하세요.

### 노트가 *실패*로 표시되는 경우
이는 보통 AI 서비스에 연결할 수 없었음을 의미합니다. 캡처된 **콘텐츠는 여전히
저장되어 있습니다** —— 노트를 열고 **↻ AI 처리 다시 시도**를 클릭하면 AI가 복구된 후
요약/태그가 채워집니다.

### 편집 후 AI 재실행
노트를 편집할 때 **재생성**할 부분을 선택할 수 있습니다 ——
**제목**, **요약**, **태그**, **카테고리**. 현재 비어 있는 필드만 미리 체크됩니다.
체크된 부분만 변경됩니다; 모두 체크 해제하면 AI를 사용하지 않고 편집만 저장합니다.

> AI 재실행은 노트의 **현재 내용** —— 즉, 편집한 내용 —— 을 기준으로 합니다. 링크/동영상/파일에서
> 캡처된 노트의 경우, 편집 후 재실행하면 **편집한 텍스트**가 사용됩니다
>(원본 콘텐츠를 다시 가져오거나 변경 사항을 덮어쓰지 않습니다).

## 5. 정리 및 탐색

- **카테고리** —— 노트에 카테고리 하나를 지정합니다(노트 양식에서 입력; 비워두면
  AI가 선택할 수 있습니다). `Work/Project A` 같은 경로를 입력하여 **하위 카테고리**를
  만듭니다. **노트** 사이드바는 카테고리를 들여쓰기된 트리로 표시합니다;
  상위 카테고리를 선택하면 하위 카테고리도 표시됩니다.
- **태그** —— AI가 태그를 제안합니다; 직접 추가할 수도 있습니다(쉼표 구분). 기존 노트의 태그를 편집하거나 삭제하려면 노트를 열어 편집 페이지의 **태그** 필드를 직접 수정하세요 —— 쉼표로 구분된 목록에서 이름을 삭제하면 제거되고, 새 이름을 입력하면 추가됩니다. 입력 중에 **기존 태그의 드롭다운 제안**이 표시되며, 방향키나 클릭으로 선택하면 쉼표와 함께 자동 입력되어 계속 이어서 입력할 수 있습니다. 태그는 사이드바와 각 노트의 클릭 가능한 칩으로 표시됩니다.
- **스마트폰에서는** 카테고리/태그 사이드바가 메뉴 바로 아래의 접을 수 있는
  **🔍 필터** 패널로 대체됩니다 —— 탭하여 동일한 카테고리 트리와
  태그 칩을 열고 노트를 필터링합니다.
- **검색** —— 검색 상자는 제목과 내용에 대해 빠른 키워드 검색을 수행합니다. 라틴 문자와 한국어·중국어·일본어를 혼합한 검색어도 지원합니다 —— 예를 들어 `Claude설치`처럼 띄어쓰기 없이 입력해도 두 언어를 모두 포함한 노트를 찾을 수 있습니다. 카테고리와 태그는 클릭 가능한 필터 단축키입니다.
- **정렬** —— **수정일**(기본), **생성일**,
  **제목**, **카테고리**, **태그**, 또는 **추가 방법**(어떻게 캡처되었는지)으로 목록을 정렬합니다.
- **페이지당 수 / 페이지네이션** —— 목록은 기본적으로 페이지당 **10**개 노트를 표시합니다;
  **페이지당** 선택기로 변경하고(10 / 25 / 50 / 100), **이전 / 다음**으로 페이지를 이동합니다.
  정렬 및 페이지 크기 선택은 기억됩니다.
- **일괄 삭제** —— 노트의 체크박스를 선택하거나 (또는 **모두 선택**) **🗑 선택 항목 삭제**를
  클릭하여 여러 노트를 한 번에 삭제합니다.
- 각 노트는 노트 페이지에 **생성** 및 **수정** 시간이 표시됩니다.

---

## 6. 캘린더 및 알림

노트 양식에서 노트에 **일정**을 추가합니다: **이벤트 날짜**, 선택적
**시작/종료 시간** 또는 **하루 종일**, 선택적 **이메일 알림**(이벤트 시간에, 또는
5분~1일 전).

- **캘린더** 페이지는 예정된 노트가 있는 월 그리드를 표시합니다; 날짜를 클릭하면
  그날의 일정이 표시됩니다. ←/오늘/→ 으로 월을 변경합니다.
- 선택한 국가의 **공휴일**이 캘린더에 표시됩니다(설정 → 캘린더 공휴일 참조).
- **알림**은 이벤트 전에 이메일로 발송됩니다(사이트에 이메일이 설정되어 있어야 합니다).

---

## 7. 노트에 질문하기

**질문** 페이지에서 자연어로 질문할 수 있습니다. Note-Aura가 노트(및 공유된 노트) 중에서
가장 관련성 높은 부분을 찾아 소스 노트로 링크되는 **인용**과 함께 답변합니다.

### 어떤 노트가 검색되나요?

| 노트 | 포함 여부 |
|------|-----------|
| 내 노트 | ✅ 예 |
| 다른 사람이 직접 공유한 노트 | ✅ 예(개인 공유를 통해) |
| 내가 속한 그룹의 노트 | ✅ 예(그룹 공유를 통해) |
| 다른 사람의 노트(공유되지 않은) | ❌ 아니요 |

> **`status = ready`**(AI 처리 완료)인 노트만 포함됩니다.
> 아직 *처리 중*이거나 *실패* 상태인 노트는 질문 결과에 나타나지 않습니다.

> 역할에서 AI를 허용하거나(또는 설정에서 자신의 API 키를 설정한 경우) 사용할 수 있습니다.
> 역할에서 AI가 꺼져 있으면 이 페이지가 숨겨질 수 있습니다.

## 8. 공유 및 그룹

### 한 사람과 노트 공유
내가 소유한 노트 열기 → **공유** → 상대방 이메일 입력, 선택적으로 **편집 가능** 체크,
**공유**. 상대방의 **공유** 탭에 표시됩니다.

### 그룹
**그룹** 페이지에서 그룹을 만들고 모든 구성원과 노트를 공유할 수 있습니다.

- **그룹 만들기**(역할 제한 적용). 당신이 **소유자 /
  그룹 관리자**가 됩니다.
- **이메일로 구성원 초대**. 상대방은 그룹 페이지에서 **수락**(또는 **거절**)해야 하는
  보류 중인 **초대**를 받습니다 —— 동의 없이 추가되는 일은 없습니다.
- **권한** —— 각 구성원은 **읽기 전용** 또는 **읽기/쓰기**입니다. 그룹 관리자는
  언제든지 전환할 수 있습니다(**쓰기 허용** / **읽기 전용으로 설정**).
- **공동 관리자** —— 그룹 관리자는 **관리자로 만들기**로 구성원을 승격할 수 있습니다.
  공동 관리자는 구성원을 초대/제거하고 권한을 관리할 수 있습니다. **소유자**만이
  그룹을 삭제할 수 있습니다.
- **나가기** —— 모든 구성원이 그룹을 나갈 수 있습니다(소유자는 대신 삭제합니다).

### 그룹에 노트 공유
내가 소유한 노트 열기 → **그룹에 공유** → 그룹 선택. 모든 구성원이 읽을 수 있으며;
읽기/쓰기 권한이 있는 구성원은 편집할 수 있습니다.

## 9. 사용자 차단

**설정 → 차단된 사용자**에서 이메일로 사용자를 차단합니다. 차단 중에는
서로 노트를 공유하거나 초대할 수 없습니다. **관리자는 차단할 수 없습니다**.
같은 곳에서 언제든지 차단을 해제할 수 있습니다.

---

## 10. 새 사용자 초대

역할에서 허용되는 경우 **설정 → 새 사용자 초대**에서 두 가지 방법으로 초대할 수 있습니다:

- **이메일로 초대** —— 이메일 주소를 입력하고 **초대 발송**을 클릭합니다. 상대방은
  등록 링크가 포함된 이메일을 받으며, 해당 링크로 가입하면 이메일 인증이 건너뛰어집니다.
  이메일 필드 옆 **언어 드롭다운**에서 초대 이메일 언어를 선택할 수 있습니다.
- **오픈 링크(이메일 불필요)** —— 이메일을 입력하지 않고 **초대 발송**을 클릭하면
  누구나 한 번 사용해 등록할 수 있는 링크가 생성됩니다. 생성 후 링크가 강조 표시된
  박스에 나타나므로 복사하여 원하는 채널(Slack, KakaoTalk 등)로 공유할 수 있습니다
  —— SMTP가 필요 없습니다.

모든 **대기 중** 초대 옆에 **링크 복사** 버튼이 있어 언제든지 등록 URL을 가져올 수
있습니다.

초대 횟수에는 제한이 있습니다(페이지에 표시됨); 제한은 역할에서 설정되며 관리자가
사용자별로 조정할 수 있습니다.

---

## 11. 이메일 → 노트

관리자가 활성화한 경우 **설정 → 이메일 → 노트**에서 개인 이메일 주소를 받습니다.
해당 주소로 **보내거나 전달**하는 모든 것이 노트가 됩니다.

1. **내 이메일 주소 생성**을 클릭합니다 —— `notes+ab12cd34@yourdomain.com` 같은
   주소를 받게 됩니다.
2. 해당 주소로 이메일을 **전송**합니다(**To** 필드에 넣음):
   - **제목**이 노트 제목이 되고, **본문**이 내용이 되며,
     AI가 요약/태그를 붙입니다.
   - 이메일 본문이 **링크만** 있는 경우 해당 페이지/동영상이 가져와집니다.
   - 역할에서 허용된 첨부 파일이 노트와 함께 저장됩니다.
3. **주소를 비밀로 유지하세요** —— 주소를 가진 사람은 누구나 계정에 노트를 추가할 수 있습니다.
   언제든지 **재생성**할 수 있습니다(이전 주소는 작동을 멈춥니다).

> 설정의 정확한 주소를 사용하고 **To** 필드에 넣어 보내며(Bcc 아님),
> `+token` 주소 —— 일반 편지함이 아닌 —— 인지 확인하세요.

## 12. 설정

- **저장소** —— 사용량과 제한(노트 + 첨부 파일 + 인라인 이미지). 제한에
  도달하면 공간을 확보하거나 관리자에게 늘려달라고 요청합니다.
- **시간대** —— 앱 내 모든 날짜 및 시간의 표시 시간대를 선택합니다(노트 생성/수정
  시간, 노트 목록, 로그 등). UTC 오프셋별로 그룹화된 40개 이상의 일반 IANA
  시간대를 지원합니다. 설정하지 않으면 UTC로 기본 설정됩니다.
- **Facebook Cookies** —— 로그인된 Facebook 브라우저에서 내보낸 Netscape `cookies.txt`를 붙여넣어
  더 풍부한 Facebook 캡처를 활성화합니다. 위의 설정 단계를 참조하세요. 개인 쿠키가 관리자 공유 쿠키보다 우선합니다.
- **AI 백엔드** —— 기본적으로 사이트의 내장 AI를 사용합니다. 자신의
  **OpenAI 호환** Base URL + API 키(OpenAI, Google Gemini 호환
  엔드포인트, OpenRouter 등)를 입력하여 자신의 계정을 사용할 수 있습니다.
  자신의 키를 사용하면 내장 일일 AI 제한이 적용되지 않습니다.
- **나의 AI 프롬프트** —— **자신의 외부 AI 서버**(위의 Base URL
  + API 키)로 실행하는 경우 해당 서버로 전송되는 제목/요약/태그/OCR 프롬프트를
  사용자 정의할 수 있습니다. (내장 AI 사용자는 관리자의 프롬프트를 사용합니다.)
- **이메일 → 노트** —— 개인 수신 주소를 생성/재생성합니다(위 참조).
- **노트 가져오기 / 내보내기** —— **전체 노트 내보내기(JSON)**로 전체 백업을 다운로드하여 나중에 다시 가져올 수 있습니다. **Markdown ZIP으로 내보내기**는 각 노트를 YAML 프론트매터(제목, 태그, 카테고리, 생성일)가 포함된 `.md` 파일로 다운로드하며, 어떤 Markdown 편집기에서도 열거나 버전 관리로 노트를 관리할 수 있습니다. **가져오기**는 JSON 백업에서 노트를 복원합니다. 제목, 본문, 요약, 태그, 카테고리, 일정이 보존됩니다; 가져온 노트는 그대로 추가됩니다(AI 재처리 없음).
- **내장 AI 사용량** —— 일일 제한이 있는 경우 오늘의 사용량이 표시됩니다.
- **캘린더 공휴일** —— 캘린더에 표시할 국가의 공휴일을 선택합니다.
- **새 사용자 초대** / **차단된 사용자** —— 위와 같습니다(여기서 자신의 보류 중인
  초대도 삭제할 수 있습니다).
- **언어** —— 상단 바에서도 변경 가능합니다.
- **사용자 가이드** —— 설정 페이지 상단의 **사용자 가이드** 링크를 클릭하면 전체 가이드를 열 수 있습니다. 관리자는 **관리 → 사용자 가이드 편집**에서 가이드 내용을 커스터마이징할 수 있습니다.

## 13. 관리자 가이드

관리자에게는 추가 **대시보드**, **사용자**, **관리** 메뉴가 표시됩니다. (처음
등록된 계정이 관리자이며; 관리자는 다른 사용자를 승격할 수 있습니다.)

### 대시보드
읽기 전용 개요:
- **합계** —— 사용자 수, 노트 수, 그룹 수, 실패한 작업 수.
- **서버 모니터** —— 가동 시간, 메모리, goroutine 수, CPU 수, Go 버전, 작업 대기열,
  데이터베이스 및 업로드 크기.
- **사용자 사용량** —— 사용자별: 역할, 노트 수, 사용된 저장소 vs 제한.
- 모든 사용자의 **최근 노트**.

사용자 사용량과 최근 노트 테이블은 **페이지네이션**이 됩니다(이전 / 다음).

### 사용자
**사용자** 탭에서 계정을 관리합니다:
- **관리자 승격/강등** —— 사용자 행의 **admin**을 체크합니다. (시스템은
  마지막 관리자를 제거하지 않습니다.)
- **역할 지정**과 **사용자별 재정의** 설정 —— 저장소(MB), 일일 AI
  제한, 초대 제한 —— 역할 기본값을 재정의합니다.
- **정지 / 정지 해제** —— 정지된 사용자는 로그인할 수 없으며 활성 세션에서 로그아웃됩니다.
  **시한부 정지**를 위한 시간 수를 입력합니다(공란 = 영구);
  계정은 시간이 지나면 자동으로 복구됩니다. (자신 또는 마지막 관리자는 정지할 수 없습니다.)
- **강제 로그아웃** —— 계정을 정지하지 않고 사용자의 모든 활성 세션을 즉시 무효화합니다.
- **잠금 해제** —— 너무 많은 실패 시도로 설정된 로그인 잠금을 해제합니다
  (계정이 현재 잠겨 있을 때만 버튼이 나타납니다).
- **삭제** —— 사용자와 모든 데이터를 삭제합니다. (자신 / 마지막 관리자는 삭제할 수 없습니다.)
- **마지막 방문** —— 각 행에 사용자가 마지막으로 활성화된 시간과 IP가 표시됩니다.
  행에는 **정지됨**(시한부인 경우 만료 시간도 표시) 및 **잠김** 배지도 표시됩니다.
- **초대** —— **초대 만들기** 양식으로 관리자가 직접 초대 링크를 생성할 수 있습니다(이메일 선택사항;
  비워두면 오픈 링크 생성). 대기 중인 초대마다 **링크 복사** 버튼이 있습니다. 모든 사용자가
  보낸 초대가 여기에 나열되며, 관리자는 어떤 것이든 **삭제**할 수 있습니다.

### 차단된 IP
**관리 → 차단된 IP** —— IP 주소와 선택적 이유를 입력하여 차단합니다.
차단된 IP에서의 요청은 세션이 확인되기 전에 HTTP 403으로 거부됩니다.
**차단 해제**를 클릭하여 차단을 제거합니다.

### 로그인 잠금
**관리 → 로그인 잠금** —— 몇 번의 연속 로그인 실패 시도가 계정을 잠그는지와
잠금이 몇 분 동안 지속되는지 설정합니다. **최대 실패 시도 횟수**를 0으로 설정하면
잠금이 비활성화됩니다. 로그인 성공 시 카운터가 자동으로 재설정됩니다; 관리자는
사용자 페이지에서 수동으로 잠금을 해제할 수도 있습니다.

### 사이트 공지
**관리 → 공지** —— 메시지를 작성하고 **공지 표시**를 체크하여 모든 페이지 상단의
황색 배너에 표시합니다. 체크 해제하면 텍스트를 잃지 않고 숨길 수 있습니다.

### 이메일 발송
**관리 → 이메일 발송** —— 일반 텍스트 이메일을 작성하여 다음 대상에 발송합니다:
- 특정 사용자(이메일 주소로),
- 역할의 모든 사용자, 또는
- 그룹의 모든 구성원.

SMTP 설정이 필요합니다; 이메일이 비활성화된 경우 페이지에 알림이 표시됩니다.

### 금지 사용자명
**관리 → 금지 사용자명** —— 이메일 주소의 로컬 파트(@ 앞 부분)를 추가하면 모든 도메인에서
해당 이름으로 등록이 금지됩니다(예: `admin` 금지 시 `admin@gmail.com`,
`admin@company.com` 등 차단). 선택적 메모를 추가할 수 있습니다.

### 금지 이메일 / 도메인
**관리 → 금지 이메일 / 도메인** —— 전체 이메일 주소를 입력하면 해당 주소를 차단하고,
도메인 이름(예: `example.com`)을 입력하면 해당 도메인의 모든 가입을 차단합니다.
선택적 메모를 추가할 수 있습니다.

### 가입
**새 사용자 가입 허용**을 전환합니다. 꺼지면 공개 가입이 닫히고 **초대**한 사람만
등록할 수 있습니다; 로그인 페이지의 *계정 만들기* 링크가 숨겨집니다.

### 역할 및 권한
역할을 만들고 각 역할에 다음을 설정합니다:
- **용량(MB)** —— 저장소 제한(0 = 무제한).
- **최대 그룹 수** —— 사용자가 **만들** 수 있는 그룹 수(0 = 없음, -1 = 무제한).
- **내장 AI(Ollama)** —— 역할이 내장 AI를 사용할 수 있는지 여부. *(자신의
  API 키가 있는 사용자는 어떻든 AI를 사용할 수 있습니다.)*
- **Ollama/일** —— 일일 내장 AI 사용 한도(0 = 무제한). 자신의 키를 사용하는
  사용자에게는 적용되지 않습니다.
- **초대 수** —— 역할이 보낼 수 있는 새 사용자 초대 수.
- **업로드** —— 역할이 업로드할 수 있는 파일 유형: **image / video /
  audio / document** 체크, 또는 추가 확장자 나열(예: `epub, csv`).

> 자신의 프롬프트 사용자 정의는 더 이상 역할 설정이 아닙니다 —— **자신의 외부 AI 서버**를
> 사용하는 모든 사용자가 설정에서 프롬프트를 편집할 수 있습니다.

### 공휴일
사용자가 캘린더에 표시할 수 있도록 공휴일 데이터를 불러옵니다:
- **온라인 가져오기** —— 국가 코드를 입력합니다(예: `US, GB, JP`) 온라인 소스에서
  가져옵니다.
- **CSV 업로드** —— 온라인 소스가 포함하지 않는 국가용(예: HK, TW,
  CN). 형식: 줄당 `date,name`, 날짜는 `YYYY-MM-DD`(헤더 행이 있어도 됩니다).

### AI 설정
- 내장 백엔드의 **각 작업에 별도의 모델 선택**(제목, 요약, 태그, 채팅/질문, OCR,
  이미지 분석, 임베딩) 및 Ollama 호스트. `config.yaml`에서 `ocr_model`로 텍스트 추출
  모델을, `image_model`로 이미지 분석 모델을 각각 지정할 수 있습니다.
- 각 작업의 글로벌 **프롬프트** 편집 —— **카테고리** 프롬프트 포함.
- **웹 링크** 및 **YouTube** 프롬프트 —— 해당 소스의 일반 프롬프트를 재정의하는 선택적
  전용 제목/요약/태그 프롬프트(공란 = 일반 프롬프트 사용).
- **Facebook cookies(Netscape cookies.txt)** —— 로그인된 Facebook 브라우저 세션에서 내보낸
  `cookies.txt`를 붙여넣어 인증된 캡처를 활성화합니다(더 풍부한
  게시물 텍스트와 동영상 자막). 비워두어도 됩니다 —— 공개 링크는 여전히
  Open-Graph 미리보기를 받습니다. 내보내기 방법은 INSTALL.md §7c를 참조하세요.

### HTTPS(TLS)
**관리 → HTTPS (TLS)** 페이지에서 **HTTPS**를 활성화 또는 비활성화하고 인증서/키 파일
경로를 설정합니다(저장할 때 쌍이 유효성 검사됩니다). 이는 `config.yaml`을 재정의하며
서버를 **재시작**한 후 적용됩니다.

### 브랜딩
**조직 로고**를 업로드하거나 헤더와 로그인 페이지에 표시되는 **문구**를 설정합니다
(로고 → 문구 → "Note-Aura").

### 이메일(SMTP 및 IMAP)
- **발신(`smtp:`)** —— 이메일 인증, 초대, 캘린더 알림에는 `config.yaml`의
  SMTP 설정이 필요합니다(host, port, username, password, from, `starttls`).
  SMTP 없이는 이러한 이메일이 건너뛰어집니다(새 계정은 자동으로 인증됩니다).
- **수신(`imap:`)** —— **이메일 → 노트**를 활성화하려면 플러스 주소 지정을
  수락하는 IMAP 사서함을 설정합니다. 세 가지 TLS 모드를 지원합니다: 암묵적 TLS(`tls: true`, 포트 993), STARTTLS(`tls: false`, 포트 143), 또는 완전 평문(`tls: false` + `starttls: false`). INSTALL.md §7d를 참조하세요.

### 웹 링크 캡처(headless)
대부분의 페이지는 **Chrome/Chromium 없이** 전체 텍스트를 자동으로 캡처합니다.
원시 HTML이 빈 껍데기인 일부 JavaScript 단일 페이지 앱(SPA)에서만 headless 모드가
필요합니다. 관리자가 `config.yaml`에서 `fetch.headless: true`를 설정한 경우에만
Chrome/Chromium이 필요합니다. 설치되지 않은 경우 headless는 자동으로 건너뛰며
다른 기능에는 영향을 주지 않습니다.
자세한 비교표와 Linux 설치 명령은 INSTALL.md §7b를 참조하세요.

### 유지보수: 업데이트 및 재설정
프로그램 옆에 두 가지 도우미 도구가 있습니다(서버에서 프로젝트 폴더 내에서 실행):

- **프로그램 업데이트, 모든 데이터 보존** —— `update.ps1`(Windows/PowerShell)은
  서버를 중지하고, 최신 코드에서 재빌드하고, 재시작합니다. 노트, 사용자,
  업로드 및 설정은 그대로 유지됩니다.
  ```powershell
  .\update.ps1
  ```
- **새 시스템으로 재설정** —— `reset.exe`는 **모든 사용자, 노트, 카테고리,
  태그, 관리자 설정을 영구적으로 삭제**하고(업로드된 이미지 모두) 설치를
  초기 상태로 되돌립니다. 서버를 먼저 중지한 후 실행하고; `RESET`을 입력하여 확인합니다.
  다음 시작 시 `config.yaml`에서 첫 번째 관리자를 다시 시드합니다.
  ```powershell
  .\reset.exe
  ```
  ⚠️ 되돌릴 수 없습니다 —— 데이터가 필요할 수 있다면 `data/note-aura.db`와 `uploads/`를
  먼저 백업하세요. INSTALL.md §9를 참조하세요.

## 14. 자주 묻는 질문

**노트가 "처리 중"에서 멈춰 있습니다.**
AI 백엔드가 바쁘거나 다운되어 있을 수 있습니다. **실패**로 표시되면 노트를 열고
**AI 처리 다시 시도**를 클릭합니다 —— 어떤 경우에도 캡처된 콘텐츠는 보존됩니다.

**노트에 내용은 있지만 요약이나 태그가 없습니다.**
AI 단계가 실패했습니다(보통 AI 호스트가 느리거나 연결 불가, 또는 큰 모델에서 대용량 콘텐츠
타임아웃). AI 호스트에 연결할 수 있는지 확인한 후 **다시 시도**하세요. 관리자는
`ai.timeout_seconds`를 올릴 수 있습니다.

**요약이 잘못된 언어로 되어 있습니다.**
노트에서(또는 캡처할 때) **요약 언어**를 설정하고 요약에 대한 AI를 재실행하세요.

**질문 페이지가 보이지 않습니다.**
역할에서 AI가 꺼져 있을 수 있습니다. 설정에서 자신의 API 키를 설정하거나
관리자에게 역할에 AI를 활성화해 달라고 요청하세요.

**링크가 한두 문장만 저장되었습니다.**
해당 페이지는 JavaScript로 렌더링되어 짧은 설명만 추출할 수 있었습니다.
관리자에게 **headless** 웹 링크 폴백을 활성화해 달라고 요청한 후 링크를
**다시 캡처**하세요(다시 시도가 아닌 재캡처가 페이지를 다시 가져옵니다).

**YouTube 동영상의 자막을 가져오지 못했습니다(설명/메타데이터만 있음).**
일부 동영상은 연령 제한이 있거나 비표준 플레이어를 사용하여 자막을 가져올 수
없습니다. Note-Aura는 여러 폴백 방법(iOS 플레이어 클라이언트, 모든 자막 트랙)을
자동으로 시도합니다. 동영상에 자막이 정말 없으면 채널명, 재생 시간, 설명만
가져옵니다. `yt-dlp`를 최신 버전으로 유지하면(`yt-dlp -U`) 호환성이 향상됩니다.

**이메일로 보낸 노트가 나타나지 않습니다.**
**설정 → 이메일 → 노트**의 `notes+<token>@…` 주소로 정확히 **To 필드에**
보내세요(Bcc가 아님) —— 일반 사서함 주소가 아닙니다. 주소를 재생성했나요? 새 주소를 사용하세요.

**다른 사람이 인터넷을 통해 사용할 수 있나요?**
배포가 공개적으로 노출된 경우 가능합니다(HTTPS 뒤에). 관리자에게 문의하세요.

**인터페이스 언어를 어떻게 변경하나요?**
상단 바의 언어 선택기를 사용합니다. 선택한 것이 없으면 브라우저를 따릅니다.

**새 가입이 닫혔습니다 —— 어떻게 가입할 수 있나요?**
관리자가 공개 등록을 꺼놓았습니다. 기존 사용자에게 **초대**해 달라고 요청하세요;
초대 링크로 등록할 수 있습니다.
